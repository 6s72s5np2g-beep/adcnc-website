### ❌ 踩雷：Cloudflare 新版 UI 把 Workers + Pages 統一了

**問題現象**：
- Cloudflare 帳號裡點 **Workers & Pages → Create**
- 點 **Connect with GitHub** 走流程
- 結果建成的是 **Worker**（看 Settings 出現 `npx wrangler deploy`、Tail Worker 等）
- 推程式碼上去後 `adcnc.com.tw` 顯示 **「Hello world!」**（Worker 預設範本）
- 點 `*.pages.dev` 子網域顯示 NXDOMAIN（沒有 Pages 子網域）

**根因**：
2024 後 Cloudflare 把 Pages 整合進 Workers，「Create application」UI 默認走 Worker 模式。即使選 GitHub repo + 識別出 Astro framework，也會建成 Worker（用 wrangler deploy 命令），不是真正的 Pages。

**解法（最後採用）**：用 wrangler CLI 從電腦直接推 Pages，繞過 UI：

```bash
npx wrangler login                    # OAuth 授權
npx wrangler pages project create adcnc-website --production-branch=main
npx wrangler pages deploy dist --project-name=adcnc-website --branch=main --commit-dirty=true
```

成功 deploy → `https://adcnc-website.pages.dev` 立刻可用。

**給未來自己**：之後改官網要更新部署，**不要走 Cloudflare UI**，直接：
```bash
cd D:\水母資料夾\銨東官網
npm run build
npx wrangler pages deploy dist --project-name=adcnc-website --branch=main --commit-dirty=true
```

### 自訂網域綁定

- 輸入 `adcnc.com.tw`
- Cloudflare 自動加 CNAME + 簽 SSL（5 分鐘內 Active）

## Stage 5：Email Routing 設定

### 流程
1. Cloudflare 帳戶首頁 → Domains → adcnc.com.tw → 左 nav **Email** → **Email Routing**
2. **Enable Email Routing** 自動加 6 筆 DNS（3 MX + 1 SPF + 2 DKIM）
3. **Routing rules** 加：
   - Custom address: `andong@adcnc.com.tw`
   - Action: Send to an email
   - Destination: `joineve0913@gmail.com`
4. **Destination addresses** 驗證 Gmail（點驗證信連結）
5. 完成後寄信到 andong@adcnc.com.tw 自動進 Gmail
